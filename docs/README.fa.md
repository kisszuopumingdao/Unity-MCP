‏`Unity MCP` یک دستیار توسعه بازی مبتنی بر هوش مصنوعی **برای Editor و Runtime** است. با این ابزار می‌توانید **Claude**، **Cursor** و **Windsurf** را از طریق پروتکل MCP به Unity وصل کنید، گردش‌کارها را خودکار کنید، کد تولید کنید و **هوش مصنوعی را مستقیماً درون بازی‌هایتان به‌کار بگیرید**.

برخلاف سایر ابزارهای مشابه، این پلاگین **درون بازی کامپایل‌شده** هم فعال است و امکان دیباگ بلادرنگ با هوش مصنوعی و تعامل مستقیم بازیکن با AI را فراهم می‌کند.

> **[💬 به سرور دیسکورد ما بپیوندید](https://discord.gg/cfbdMZX99G)** — سوال بپرسید به انگلیسی البته، پروژه‌هایتان را به نمایش بگذارید و با دیگر توسعه‌دهندگان در ارتباط باشید!

## ![AI Game Developer — Unity MCP](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/img/promo/hazzard-features.svg?raw=true)

- ✔️ **عامل‌های هوش مصنوعی** — از بهترین عامل‌های **Anthropic**، **OpenAI**، **Microsoft** یا هر ارائه‌دهنده دیگری استفاده کنید؛ بدون وابستگی به یک پلتفرم خاص
- ✔️ **ابزارها** — مجموعه گسترده‌ای از [MCP Tools](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/default-mcp-tools.md) پیش‌فرض برای کار در Unity Editor
- ✔️ **مهارت‌ها** — تولید خودکار مهارت برای هوش مصنوعی بر اساس سیستم‌عامل، نسخه Unity و پلاگین‌های نصب‌شده در پروژه
- ✔️ **کد و تست** — مکانیک‌های بازی را توسعه داده و با کمک عامل‌های هوش مصنوعی آزمایش کنید
- ✔️ **Runtime (درون بازی)** — استفاده مستقیم از LLM درون بازی کامپایل‌شده، برای رفتار پویای NPC یا دیباگ حین اجرا
- ✔️ **پشتیبانی از دیباگ** — اجازه دهید هوش مصنوعی مشکلات پروژه را شناسایی و برطرف کند
- ✔️ **مکالمه طبیعی** — درست مثل صحبت با یک انسان، با هوش مصنوعی گفتگو کنید
- ✔️ **استقرار انعطاف‌پذیر** — هم به صورت محلی (stdio) و هم از راه دور (http) از طریق پیکربندی قابل استفاده است
- ✔️ **قابل توسعه** — [ابزارهای سفارشی را مستقیم در کد پروژه‌تان بسازید](#add-custom-tool)

# شروع سریع

در سه مرحله راه‌اندازی کنید:

1. **[نصب پلاگین](#step-1-install-unity-mcp-plugin)** — فایل `.unitypackage` نصب‌کننده را دانلود کنید یا دستور `openupm add com.ivanmurzak.unity.mcp` را اجرا کنید
   > **روش جایگزین:** ‏`npx unity-mcp-cli install-plugin ./MyUnityProject` — راهنمای [مستندات CLI](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md)
2. **[انتخاب عامل هوش مصنوعی](#step-2-install-ai-agent)** — Claude Code، Claude Desktop، GitHub Copilot، Cursor یا هر گزینه پشتیبانی‌شده دیگری
3. **[پیکربندی عامل هوش مصنوعی](#step-3-configure-ai-agent)** — در Unity مسیر `Window/AI Game Developer` را باز کرده و روی **Auto-generate skills** (پیشنهادی) یا **Configure MCP** کلیک کنید
   ![Setup AI Skills](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/img/editor/setup-skills.jpg?raw=true)
   > **روش جایگزین:** ‏`npx unity-mcp-cli setup-skills claude-code ./MyUnityProject` — راهنمای [مستندات CLI](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md)

همین! از هوش مصنوعی بخواهید *«سه مکعب در یک دایره با شعاع ۲ بساز»* و نتیجه را ببینید. ✨

# نصب

<a id="step-1-install-unity-mcp-plugin"></a>
## مرحله ۱: نصب `Unity MCP Plugin`

**⚠️ پیش‌نیازها:**

> [!IMPORTANT]
> **مسیر پروژه نباید فاصله داشته باشد**
>
> - ✅ ‏`C:/MyProjects/MyProject`
> - ❌ ‏`C:/My Projects/MyProject`
> - ❌ ‏`C:/My Projects/My Project`
> - ❌ ‏`C:/MyProjects/My Project`

### گزینه ۱ — نصب‌کننده

- **[⬇️ دانلود نصب‌کننده](https://github.com/IvanMurzak/Unity-MCP/releases/latest/download/AI-Game-Dev-Installer.unitypackage)**
- **📂 فایل را در پروژه Unity ایمپورت کنید**
  > - می‌توانید روی فایل دوبار کلیک کنید — Unity آن را به‌طور خودکار باز می‌کند
  > - یا اینکه: ابتدا Unity Editor را باز کنید، سپس روی `Assets/Import Package/Custom Package` کلیک کرده و فایل را انتخاب کنید

### گزینه ۲ — CLI (پیشنهادی)

پلاگین را از طریق [`unity-mcp-cli`](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md) نصب کنید — بدون نیاز به باز کردن Unity Editor:

```bash
# ۱.۱ نصب unity-mcp-cli
npm install -g unity-mcp-cli

# ۱.۲ (اختیاری) نصب Unity
unity-mcp-cli install-unity

# ۱.۳ (اختیاری) ساخت پروژه Unity جدید
unity-mcp-cli create-project ./MyUnityProject

# ۲. نصب «AI Game Developer» در پروژه Unity
unity-mcp-cli install-plugin ./MyUnityProject

# ۳. ورود به سرور ابری
unity-mcp-cli login ./MyUnityProject

# ۴. باز کردن پروژه Unity (اتصال خودکار و تولید مهارت‌ها)
unity-mcp-cli open ./MyUnityProject

# ۵. صبر کنید تا Unity Editor آماده شود
unity-mcp-cli wait-for-ready ./MyUnityProject
```

> برای فهرست کامل دستورات، [مستندات CLI](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md) را ببینید.

<a id="step-3-configure-ai-agent"></a>
## مرحله ۳: پیکربندی `عامل هوش مصنوعی`

### پیکربندی خودکار

- پروژه Unity را باز کنید
- مسیر `Window/AI Game Developer` را باز کنید
- گزینه ۱: روی `Auto-generate Skills` کلیک کنید **(پیشنهادی)**
- گزینه ۲: روی `Configure` برای Model Context Protocol (MCP) کلیک کنید

> اگر MCP client شما در لیست نبود، از JSON خامی که در پنجره نمایش داده می‌شود استفاده کرده و آن را به تنظیمات MCP client خود اضافه کنید. برای این کار به راهنمای مخصوص همان MCP client مراجعه کنید.

### پیکربندی دستی

اگر پیکربندی خودکار به هر دلیلی کار نکرد، می‌توانید از JSON موجود در پنجره `AI Game Developer (Unity-MCP)` برای پیکربندی دستی هر `MCP Client` استفاده کنید.

#### پیکربندی از طریق خط فرمان

**۱. پلتفرم و دستور `<command>` مناسب را انتخاب کنید:**

| پلتفرم | `<command>` |
| --- | --- |
| ‏Windows x64 | `"<unityProjectPath>/Library/mcp-server/win-x64/unity-mcp-server.exe" port=<port> client-transport=stdio` |
| ‏Windows x86 | `"<unityProjectPath>/Library/mcp-server/win-x86/unity-mcp-server.exe" port=<port> client-transport=stdio` |
| ‏Windows arm64 | `"<unityProjectPath>/Library/mcp-server/win-arm64/unity-mcp-server.exe" port=<port> client-transport=stdio` |
| ‏MacOS Apple Silicon | `"<unityProjectPath>/Library/mcp-server/osx-arm64/unity-mcp-server" port=<port> client-transport=stdio` |
| ‏MacOS Apple Intel | `"<unityProjectPath>/Library/mcp-server/osx-x64/unity-mcp-server" port=<port> client-transport=stdio` |
| ‏Linux x64 | `"<unityProjectPath>/Library/mcp-server/linux-x64/unity-mcp-server" port=<port> client-transport=stdio` |
| ‏Linux arm64 | `"<unityProjectPath>/Library/mcp-server/linux-arm64/unity-mcp-server" port=<port> client-transport=stdio` |

**۲.** ‏`<unityProjectPath>` را با مسیر کامل پروژه Unity جایگزین کنید

**۳.** ‏`<port>` را با پورتی که در تنظیمات AI Game Developer مشخص شده جایگزین کنید

**۴.** سرور MCP را از طریق خط فرمان اضافه کنید

# نمونه‌های گردش‌کار با هوش مصنوعی

در `عامل هوش مصنوعی` خود مستقیماً با LLM صحبت کنید و هر کاری که می‌خواهید از آن بخواهید. هرچه وظیفه یا ایده خود را واضح‌تر توصیف کنید، نتیجه بهتری می‌گیرید.

برخی `عامل‌های هوش مصنوعی` به شما امکان می‌دهند مدل LLM مورد استفاده را تغییر دهید. به این قابلیت توجه داشته باشید، چراکه برخی مدل‌ها ممکن است عملکرد به‌مراتب بهتری نسبت به بقیه داشته باشند.

**نمونه درخواست‌ها:**

```text
ساختار سلسله‌مراتبی صحنه‌ام را برایم توضیح بده
```

```text
سه کره روی هم بساز
```

```text
یک متریال طلایی متالیک بساز و آن را به یک GameObject کره جدید اضافه کن
```

> اگر از VS Code با Copilot استفاده می‌کنید، مطمئن شوید حالت `Agent` فعال باشد.

## غیرفعال کردن اعلان به‌روزرسانی برای کل تیم

وقتی نسخه جدیدتری از پلاگین در OpenUPM موجود باشد، یک پاپ‌آپ هنگام راه‌اندازی Editor نمایش داده می‌شود. به‌طور پیش‌فرض این پاپ‌آپ برای هر عضو تیم جداگانه نمایش می‌یابد تا زمانی که هرکدام روی *«دیگر نشان نده»* کلیک کنند؛ این تنظیم به صورت per-user روی دستگاه هر فرد ذخیره می‌شود.

در پروژه‌هایی که یک مهندس مسئول مدیریت نسخه پلاگین است، می‌توانید این پاپ‌آپ را برای **همه اعضای تیم** غیرفعال کنید. کافی است وارد **Edit ▸ Project Settings ▸ AI Game Developer** شوید و گزینه *«Disable update notifications for the entire team»* را فعال کنید. این تنظیم در فایل `ProjectSettings/AI-Game-Developer-UpdateSettings.asset` ذخیره می‌شود — آن را commit کنید تا همه اعضای تیم پس از pull کردن، دیگر این پاپ‌آپ را نبینند.

همین گزینه از مسیر **Tools ▸ AI Game Developer ▸ Updates ▸ Disable Update Notifications (Team)** در نوار منو هم قابل دسترسی است.

## ویژگی‌های پیشرفته برای LLM

Unity MCP ابزارهای پیشرفته‌ای دارد که کمک می‌کنند LLM سریع‌تر و دقیق‌تر کار کند، از بروز خطا جلوگیری شود و در صورت بروز مشکل، خودکار اصلاح صورت گیرد. همه این موارد با هدف رسیدن مؤثر به خواسته‌های شما طراحی شده‌اند.

### قابلیت‌های اصلی

- ✔️ **ابزارهای آماده برای عامل** — هر چیزی که نیاز دارید را در ۱ تا ۲ قدم پیدا کنید
- ✔️ **کامپایل فوری** — کامپایل و اجرای کد #C با استفاده از `Roslyn` برای تکرار سریع‌تر
- ✔️ **دسترسی کامل به asset** — خواندن و نوشتن assetها و اسکریپت‌های #C
- ✔️ **بازخورد هوشمند** — پیام‌های خطا و موفقیت واضح و مفهوم، برای درک درست مشکلات

### ویژگی‌های مبتنی بر Reflection

- ✔️ **ارجاع به اشیاء** — دسترسی به اشیاء موجود برای تولید فوری کد #C
- ✔️ **دسترسی به داده‌های پروژه** — مشاهده کامل تمام داده‌های پروژه در قالبی قابل خواندن
- ✔️ **ویرایش دانه‌ریز** — تغییر هر قطعه داده در پروژه به صورت دقیق
- ✔️ **کشف متد** — یافتن هر متدی در کل کدبیس، حتی در فایل‌های DLL کامپایل‌شده
- ✔️ **اجرای متد** — فراخوانی هر متدی در کل کدبیس
- ✔️ **پارامترهای پیشرفته** — ارسال هر نوع property به فراخوانی متد، حتی ارجاع به اشیاء زنده در حافظه
- ✔️ **Unity API زنده** — API یونیتی همیشه به‌روز در دسترس است — حتی پس از آپدیت Unity
- ✔️ **خود-مستندساز** — دسترسی به توضیحات قابل‌خواندن هر `class`، `method` یا `property` از طریق ویژگی‌های `Description`

# سفارشی‌سازی ابزارها

**[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** به صاحبان پروژه امکان می‌دهد `MCP Tool`، `MCP Resource` و `MCP Prompt` سفارشی بسازند. سرور MCP داده‌ها را از `Unity MCP Plugin` دریافت کرده و در اختیار client قرار می‌دهد. هر عنصر جدیدی که در زنجیره MCP تعریف شود، برای LLM قابل رویت خواهد بود و ممکن است در جریان کار از آن استفاده کند.

<a id="add-custom-tool"></a>
## افزودن `Tool` سفارشی

برای افزودن یک `Tool` سفارشی، به موارد زیر نیاز دارید:

1. یک کلاس با ویژگی `AiToolType`
2. یک متد در آن کلاس با ویژگی `AiTool`
3. *(اختیاری)* افزودن ویژگی `Description` به هر آرگومان متد تا LLM بداند هر پارامتر چیست
4. *(اختیاری)* استفاده از `string? optional = null` با علامت `?` و مقدار پیش‌فرض، تا پارامتر به عنوان `optional` شناخته شود

> توجه کنید که خط ‏`MainThread.Instance.Run(() =>` اجرای کد را به thread اصلی منتقل می‌کند که برای استفاده از Unity API الزامی است. اگر نیازی به thread اصلی ندارید، از آن پرهیز کنید تا کارایی بهتری داشته باشید.

## افزودن `MCP Prompt` سفارشی

‏`MCP Prompt` به شما امکان می‌دهد دستورالعمل‌های از پیش‌نوشته را به مکالمه با LLM تزریق کنید. از دو نقش فرستنده پشتیبانی می‌کند: User و Assistant. این ساده‌ترین روش برای دادن دستورالعمل‌های خاص به LLM است. می‌توانید promptها را بر اساس داده‌های پویا تولید کنید و اطلاعات مرتبط را درون آن‌ها بگنجانید.

# استفاده در Runtime (درون بازی)

از **[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** مستقیماً درون بازی یا اپلیکیشن خود استفاده کنید. می‌توانید از Tools، Resources یا Prompts بهره بگیرید. به‌طور پیش‌فرض هیچ ابزار آماده‌ای وجود ندارد و باید ابزارهای سفارشی خود را پیاده‌سازی کنید.

## نمونه: ربات شطرنج مبتنی بر هوش مصنوعی

فرض کنید یک بازی شطرنج کلاسیک دارید و می‌خواهید منطق ربات را به LLM بسپارید. ربات باید بر اساس قوانین بازی حرکت مناسب را انتخاب کند.

## چرا استفاده در Runtime اهمیت دارد؟

کاربردهای زیادی وجود دارد. برای مثال، در یک بازی شطرنج با ربات، می‌توانید تصمیم‌گیری ربات را با چند خط کد به LLM واگذار کنید و نیازی به پیاده‌سازی منطق پیچیده bot نداشته باشید.

# راه‌اندازی `MCP Server` در Unity

سرور **[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** از گزینه‌های راه‌اندازی متعدد و استقرار Docker پشتیبانی می‌کند. هر دو پروتکل انتقال `streamableHttp` و `stdio` پشتیبانی می‌شوند. اگر قصد دارید Unity MCP Server را سفارشی‌سازی کنید یا روی ابر مستقر کنید، این بخش مناسب شماست. [بیشتر بخوانید...](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/mcp-server.md)

## متغیرها

صرف نظر از روش راه‌اندازی، تمام گزینه‌ها از پیکربندی سفارشی از طریق Environment Variables و Command Line Arguments پشتیبانی می‌کنند. اگر فقط می‌خواهید سرور را راه‌اندازی کنید، مقادیر پیش‌فرض کافی است و نیازی به تنظیم دستی ندارید. فقط مطمئن شوید Unity Plugin هم مقادیر پیش‌فرض یکسانی دارد، به‌ویژه `--port` که باید در هر دو طرف برابر باشد.

| متغیر محیطی | آرگومان خط فرمان | توضیح |
| --- | --- | --- |
| `MCP_PLUGIN_PORT` | `--port` | پورت اتصال **Client** ← **Server** ← **Plugin** (پیش‌فرض: ۸۰۸۰) |
| `MCP_PLUGIN_CLIENT_TIMEOUT` | `--plugin-timeout` | تایم‌اوت اتصال **Plugin** به **Server** (میلی‌ثانیه) (پیش‌فرض: ۱۰۰۰۰) |
| `MCP_PLUGIN_CLIENT_TRANSPORT` | `--client-transport` | نوع انتقال **Client** به **Server**‏: `stdio` یا `streamableHttp` (پیش‌فرض: `streamableHttp`) |
| `MCP_AUTHORIZATION` | `--authorization` | حالت احراز هویت برای اتصالات **Client** ورودی: `none` یا `required` (پیش‌فرض: `none`) |
| `MCP_PLUGIN_TOKEN` | `--token` | توکن Bearer مورد نیاز از **Client** هنگامی که `--authorization=required` تنظیم شده (پیش‌فرض: تنظیم‌نشده) |
| `MCP_PLUGIN_IDLE_TIMEOUT_SECONDS` | `--idle-timeout-seconds` | خاموش کردن سرور پس از این تعداد ثانیه بی‌اتصالی (پیش‌فرض: ۶۰۰) |

> آرگومان‌های خط فرمان از پیشوند `-` تکی (مثلاً `-port`) و حتی بدون پیشوند (مثلاً `port`) هم پشتیبانی می‌کنند.

> **راهنمای انتخاب transport:** از `stdio` استفاده کنید وقتی MCP client مستقیماً باینری سرور را اجرا می‌کند (رایج‌ترین حالت برای استفاده محلی). از `streamableHttp` استفاده کنید وقتی سرور به عنوان یک process مستقل یا در Docker/ابر اجرا می‌شود و اتصال از طریق HTTP برقرار می‌گردد.

## متغیرهای پلاگین

Unity MCP Plugin در هنگام راه‌اندازی، متغیرهای محیطی (و آرگومان‌های خط فرمان) زیر را می‌خواند تا مقادیر فایل پیکربندی ذخیره‌شده را بازنویسی کند. این بازنویسی‌ها در زمان اجرا اعمال می‌شوند؛ در اولین اجرا یا هنگام تولید توکن جدید احراز هویت، مقادیر جدید **در فایل پیکربندی ذخیره می‌شوند**. در اجراهای بعدی، بازنویسی‌ها فقط در حافظه اعمال شده و به‌طور خودکار ذخیره نمی‌شوند. استثنا متغیر `UNITY_MCP_TOOLS` است که از `[JsonIgnore]` استفاده می‌کند و **هرگز در فایل ذخیره نمی‌شود** — صرفاً یک متغیر runtime است.

> آرگومان‌های خط فرمان بر متغیرهای محیطی اولویت دارند و هر دو مقدار فایل پیکربندی ذخیره‌شده را بازنویسی می‌کنند. پرچم‌های کوتاه `-url`، `-token` و `-auth` نام مستعار هستند؛ نام‌های کامل `-UNITY_MCP_*` هم پذیرفته می‌شوند.

## Docker 📦

مطمئن شوید Docker نصب شده است. اگر روی ویندوز هستید، Docker Desktop را هم راه‌اندازی کنید.

[راهنمای کامل پیکربندی Docker را بخوانید](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/DOCKER_DEPLOYMENT.md).

# نحوه کار معماری Unity MCP

**[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** به عنوان یک پل بین LLM و Unity عمل می‌کند. ابزارهای Unity را به LLM معرفی کرده و توضیح می‌دهد، و LLM با درک این رابط، ابزارها را در پاسخ به درخواست‌های کاربر به‌کار می‌گیرد.

**[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** از طریق پنجره `AI Connector` به client‌های LLM مانند [Claude](https://claude.ai/download) یا [Cursor](https://www.cursor.com/) وصل می‌شود. client‌های سفارشی هم پشتیبانی می‌شوند.

این سیستم به شدت قابل توسعه است — می‌توانید `MCP Tool`، `MCP Resource` یا `MCP Prompt` سفارشی را مستقیم در کدبیس پروژه Unity خود تعریف کنید و قابلیت‌های جدیدی برای client‌های هوش مصنوعی یا اتوماسیون‌های شما نمایش دهید. همین ویژگی، Unity MCP را به یک پایه انعطاف‌پذیر برای ساخت گردش‌کارهای پیشرفته، نمونه‌سازی سریع و یکپارچه‌سازی قابلیت‌های هوش مصنوعی در فرآیند توسعه تبدیل می‌کند.

## MCP چیست؟

MCP مخفف Model Context Protocol است. به زبان ساده، چیزی شبیه به `USB Type-C` برای دنیای هوش مصنوعی است — به‌ویژه برای LLMها. به مدل زبانی یاد می‌دهد چطور با ابزارهای خارجی کار کند؛ مثل Unity Engine در این پروژه، یا حتی متدهای سفارشی در کد شما. [مستندات رسمی](https://modelcontextprotocol.io/)

## `عامل هوش مصنوعی` چیست؟

یک اپلیکیشن با پنجره چت است که ممکن است عامل‌های هوشمند داخلی و ابزارهای پیشرفته MCP داشته باشد. در عمل، یک MCP Client خوب نقش پنجاه‌درصد موفقیت هوش مصنوعی در انجام یک وظیفه را دارد. به همین دلیل انتخاب درست آن اهمیت زیادی دارد.

## `MCP Server` چیست؟

یک واسط بین `MCP Client` و «چیز دیگری» است — در این مورد خاص، همان Unity Engine. این پروژه خود شامل `MCP Server` می‌شود.

## `MCP Tool` چیست؟

‏`MCP Tool` یک تابع یا متد است که LLM می‌تواند برای تعامل با Unity فراخوانی کند. این ابزارها پل بین درخواست‌های زبان طبیعی و عملیات واقعی Unity هستند. وقتی از هوش مصنوعی می‌خواهید «یک مکعب بساز» یا «رنگ متریال را عوض کن»، از همین MCP Tools برای انجام آن استفاده می‌کند.

**ویژگی‌های کلیدی:**

- **توابع قابل اجرا** که عملیات مشخصی انجام می‌دهند
- **پارامترهای نوع‌دار** با توضیحاتی که به LLM کمک می‌کنند بداند چه داده‌ای ارسال کند
- **مقادیر برگشتی** که نتیجه موفقیت یا شکست عملیات را اعلام می‌کنند
- **آگاه از thread** — می‌توانند روی thread اصلی برای Unity API یا thread پس‌زمینه برای پردازش سنگین اجرا شوند

### چه زمانی از `Tool` استفاده کنیم

- **خودکارسازی کارهای تکراری** — ابزارهایی بسازید برای کارهایی که مکرراً انجام می‌دهید
- **عملیات پیچیده** — چندین فراخوانی Unity API را در یک ابزار ساده ترکیب کنید
- **گردش‌کارهای خاص پروژه** — ابزارهایی بسازید که ساختار و قراردادهای پروژه‌تان را می‌شناسند
- **وظایف مستعد خطا** — ابزارهایی با اعتبارسنجی و مدیریت خطای داخلی بسازید
- **منطق بازی سفارشی** — سیستم‌های بازی خود را به هوش مصنوعی نمایش دهید تا محتوای پویا تولید کند

**مثال‌ها:**

- ساخت و پیکربندی GameObject با کامپوننت‌های خاص
- پردازش دسته‌ای asset‌ها (تکسچر، متریال، prefab)
- راه‌اندازی نورپردازی و افکت‌های post-processing
- تولید procedural هندسه سطح یا جانمایی اشیاء
- تنظیم پارامترهای فیزیک یا لایه‌های collision

## `MCP Resource` چیست؟

‏`MCP Resource` دسترسی فقط‌خواندنی به داده‌های پروژه Unity را فراهم می‌کند. برخلاف MCP Tools که کاری انجام می‌دهند، Resources به LLM امکان می‌دهند وضعیت جاری، asset‌ها و پیکربندی پروژه را بررسی و درک کنند. آن‌ها را مثل «حسگرهایی» تصور کنید که به هوش مصنوعی زمینه و آگاهی درباره پروژه‌تان می‌دهند.

**ویژگی‌های کلیدی:**

- **دسترسی فقط‌خواندنی** به داده‌های پروژه و اشیاء Unity
- **اطلاعات ساختاریافته** در قالبی که LLM می‌تواند پردازش کند
- **داده‌های بلادرنگ** که وضعیت فعلی پروژه را منعکس می‌کنند
- **آگاهی زمینه‌ای** که به هوش مصنوعی کمک می‌کند تصمیم‌های بهتری بگیرد

### چه زمانی از `MCP Resource` استفاده کنیم

- **تحلیل پروژه** — اجازه دهید هوش مصنوعی ساختار، asset‌ها و نحوه سازماندهی پروژه را بشناسد
- **کمک در دیباگ** — اطلاعات وضعیت فعلی را برای عیب‌یابی فراهم کنید
- **پیشنهادهای هوشمند** — با دادن زمینه مناسب به هوش مصنوعی، توصیه‌های دقیق‌تری دریافت کنید
- **تولید مستندات** — بر اساس وضعیت پروژه به‌طور خودکار مستند تولید کنید
- **مدیریت asset** — به هوش مصنوعی کمک کنید بداند چه asset‌هایی موجودند و چه ویژگی‌هایی دارند

**مثال‌ها:**

- نمایش سلسله‌مراتب صحنه و ویژگی‌های GameObject
- فهرست‌کردن متریال‌ها، تکسچرها و تنظیمات آن‌ها
- نمایش وابستگی‌های اسکریپت و روابط کامپوننت‌ها
- نمایش تنظیمات نورپردازی و پیکربندی render pipeline
- ارائه اطلاعات درباره منابع صوتی، انیمیشن‌ها و سیستم‌های ذرات

## `MCP Prompt` چیست؟

‏`MCP Prompt` به شما امکان می‌دهد دستورالعمل‌های از پیش‌تعریف‌شده را به مکالمه با LLM تزریق کنید. این‌ها قالب‌هایی هستند که می‌توانند زمینه، راهنما یا دانش تخصصی را برای هدایت رفتار هوش مصنوعی فراهم کنند. Promptها می‌توانند متن ثابت باشند یا بر اساس وضعیت جاری پروژه به صورت پویا تولید شوند.

**ویژگی‌های کلیدی:**

- **راهنمای زمینه‌ای** که بر نحوه پاسخ‌دهی هوش مصنوعی تأثیر می‌گذارد
- **نقش‌محور** — می‌تواند شخصیت‌های مختلف را شبیه‌سازی کند (درخواست‌های User یا دانش Assistant)
- **محتوای پویا** — می‌تواند شامل داده‌های بلادرنگ پروژه باشد
- **قالب‌های قابل استفاده مجدد** برای سناریوها و گردش‌کارهای رایج

### چه زمانی از `MCP Prompt` استفاده کنیم

- **انتقال دانش تخصصی** — بهترین روش‌ها و استانداردهای کدنویسی خاص پروژه را به هوش مصنوعی یاد دهید
- **تعریف قراردادهای کدنویسی** — قراردادهای نام‌گذاری، الگوهای معماری و سبک کد را مشخص کنید
- **ارائه زمینه درباره ساختار پروژه** — توضیح دهید پروژه چگونه سازمان یافته و چرا این‌طور طراحی شده
- **اشتراک‌گذاری راهنمای گردش‌کار** — مراحل گام‌به‌گام برای کارهای رایج فراهم کنید
- **تزریق دانش ویژه** — اطلاعاتی درباره قابلیت‌های خاص Unity، asset‌های شخص ثالث یا سیستم‌های سفارشی اضافه کنید

**مثال‌ها:**

- «همیشه برای متدهای public از PascalCase و برای فیلدهای private از camelCase استفاده کن»
- «این پروژه از یک سیستم رویداد سفارشی در مسیر ‏`Scripts/Events/` استفاده می‌کند»
- «هنگام ساختن عناصر UI، همیشه آن‌ها را به Canvas موجود در ‏`Scene/UI/MainCanvas` اضافه کن»
- «کارایی حیاتی است — برای اشیائی که مکرراً ساخته می‌شوند، object pooling را ترجیح بده»
- «این پروژه از اصول SOLID پیروی می‌کند — هر تصمیم معماری باید توجیه داشته باشد»

# حذف نصب 🧹

برای حذف کامل افزونه از پروژه‌ی Unity خود:

1. منوی **Window ▸ Package Manager** را باز کنید، بسته‌ی **AI Game Developer — MCP** را انتخاب کرده و روی **Remove** کلیک کنید.
2. Unity را ببندید و سپس پوشه‌ی `Assets/Plugins/NuGet` (به همراه فایل `Assets/Plugins/NuGet.meta` آن) را حذف کنید.

# مشارکت 💙💛

مشارکت‌ها بسیار ارزشمند هستند. ایده‌هایتان را بیاورید و با هم توسعه بازی را ساده‌تر از همیشه کنیم! ایده‌ای برای `Tool` یا ویژگی جدید دارید، یا باگی پیدا کرده‌اید و می‌دانید چطور رفعش کنید؟

**اگر این پروژه برایتان مفید بوده، حتماً ستاره بزنید 🌟**

1. 👉 [مستندات توسعه را مطالعه کنید](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/dev/Development.md)
2. 👉 [پروژه را Fork کنید](https://github.com/IvanMurzak/Unity-MCP/fork)
3. Fork را clone کرده و پوشه `./Unity-MCP-Plugin` را در Unity باز کنید
4. قابلیت جدید را پیاده‌سازی کنید، commit کرده و به GitHub push کنید
5. یک Pull Request به مخزن اصلی [Unity-MCP](https://github.com/IvanMurzak/Unity-MCP/compare) روی شاخه `main` ایجاد کنید
